Promptengineering

Stap 02

De vier bouwstenen

Een goede prompt bevat altijd vier elementen. Ontbreekt er één, dan daalt de kwaliteit van het antwoord merkbaar.

Rol
Geef de AI een identiteit. “Jij bent een ervaren leraar Nederlands” levert betere resultaten dan een anonieme opdracht.
Taak
Wees specifiek. “Maak 10 meerkeuzevragen over de Tweede Wereldoorlog voor 5de jaar” is concreet. “Schrijf een quiz” is vaag.
Context
Geef relevante achtergrond mee: leerjaar, vak, niveau, doelstelling, beschikbare tijd. Context = kwaliteit.
Formaat
Beschrijf hoe de output er moet uitzien: lijstje, tabel, HTML-pagina. Zonder formaat krijg je proza.

Gouden regel: een vaag prompt geeft een vaag antwoord. Neem 30 seconden extra om je prompt te structureren — het resultaat is tien keer beter.

Stap 02

Itereren & verfijnen

Het eerste antwoord is zelden het beste. AI is een gesprek, geen zoekopdracht.

  • 01Stuur je eerste prompt — bekijk het resultaat kritisch.
  • 02Vraag door: "Maak het concreter voor 4de jaar" of "Voeg meer voorbeelden toe".
  • 03Pas de context aan: "Stel dat leerlingen dit thema nog nooit gezien hebben".
  • 04Vraag alternatieven: "Geef me drie versies op verschillende niveaus".
  • 05Verfijn het formaat: "Zet dit in een tabel met kolommen voor niveau, activiteit en tijdsduur".

Metaprompt: vraag de AI om jouw prompt te verbeteren voor je hem verstuurt: "Verbeter deze prompt zodat ik een beter antwoord krijg: [jouw prompt]"

Stap 02

Gamificatie-elementen

AI genereert speelse oefeningen die leerlingen motiveren. Van puntensystemen tot escape rooms — alles is mogelijk.

Puntensysteem
Leerlingen verdienen punten voor correct antwoorden. AI schrijft de regels en de feedbackteksten.
Escape room
Een reeks raadsels die leerlingen stap voor stap oplossen. AI genereert het verhaal en de puzzels.
Niveaubadges
Drie niveaus: beginner, gevorderd, expert. AI past de moeilijkheid automatisch aan.
Hints-systeem
Leerlingen kunnen hints opvragen die punten kosten. AI schrijft de gegradueerde hints.
Stap 02

Speelbare apps maken

Gemini schrijft HTML, CSS en JavaScript in één bestand. Jij plakt het online — klaar. Geen programmeerkennis nodig.

Interactieve quiz

Timer, meerkeuze, directe feedback, scoreteller. Solo of klassikaal op het grote scherm.

Prompt — Interactieve quiz
Schrijf een volledige werkende HTML-pagina voor een interactieve quiz over [onderwerp] voor [graad + vak]. Vereisten: • 10 meerkeuzevragen met 4 antwoorden elk • Timer van 20 seconden per vraag • Directe feedback: groen/rood + korte uitleg • Scoreteller zichtbaar tijdens het spel • Eindscherm met score en motiverende boodschap Gebruik alleen HTML, CSS en JavaScript in één bestand.

Matching-app

Begrippen aan definities koppelen, tijdlijnen ordenen, categorieën sorteren.

Prompt — Matching-app
Schrijf een volledige werkende HTML-pagina voor een matching-oefening over [onderwerp] voor [graad + vak]. Vereisten: • 10 begrippen aan hun definitie koppelen via klikken • Correcte match: groen + verdwijnt uit de lijst • Foute match: rode flits + opnieuw proberen • Progressiebalk die vordert bij elke correcte match Gebruik alleen HTML, CSS en JavaScript in één bestand.

Keuzeverhaal

Leerling maakt keuzes die het verhaal sturen. Meerdere eindes, reflectievraag achteraf.

Prompt — Keuzeverhaal
Schrijf een volledige werkende HTML-pagina voor een interactief keuzeverhaal over [thema/dilemma] voor [graad + vak]. Vereisten: • Verhalende inleiding die de situatie schetst • Minstens 4 keuzemomenten met 2-3 opties elk • Minstens 2 mogelijke eindes • Reflectievraag na elk einde Gebruik alleen HTML, CSS en JavaScript in één bestand.

Simulatie / rekentool

Sliders en invoervelden die live berekeningen uitvoeren. Ideaal voor wiskunde, fysica, economie.

Prompt — Simulatie
Schrijf een volledige werkende HTML-pagina voor een interactieve simulatie van [concept/proces] voor [graad + vak]. Vereisten: • Invoervelden of sliders voor de variabelen • Live berekening en grafische weergave • Uitleg bij elk resultaat • Resetknop om opnieuw te beginnen Gebruik alleen HTML, CSS en JavaScript in één bestand.

Oefenreeks

Stijgende moeilijkheid, progressiebalk, hint-knop. Werkt voor elk vak.

Prompt — Oefenreeks
Schrijf een volledige werkende HTML-pagina voor een stapsgewijze oefenreeks over [leerstof] voor [graad + vak]. Vereisten: • 12 oefeningen op 3 niveaus (4/4/4) • Progressiebalk bovenaan • Hint-knop per vraag (kost 1 punt) • Eindscherm met score en niveau bereikt Gebruik alleen HTML, CSS en JavaScript in één bestand.

Flitskaarten

Flip-animatie, "Ik kende het / Nog oefenen", shuffle. Perfect voor woordenschat en begrippen.

Prompt — Flitskaarten
Schrijf een volledige werkende HTML-pagina voor een flitskaarten-app met de woordenschat van [thema/hoofdstuk] voor [graad + vak]. Vereisten: • 20 flitskaarten: voorkant = woord, achterkant = definitie + voorbeeld • Flip-animatie bij klikken • Knoppen: "Ik kende het" / "Nog oefenen" • Shuffle-knop en voortgangsweergave Gebruik alleen HTML, CSS en JavaScript in één bestand.

Multiplayer? Niet mogelijk met een HTML-bestand. Gebruik Kahoot! of Quizizz voor realtime klassikale quizzen. Laat Gemini de vragen schrijven en plak ze erin.

Stap 02

Publiceren via Google Sites

Gratis, werkt met je schoolaccount van Don Bosco. Klaar in minder dan 5 minuten.

  • 1.Ga naar sites.google.com → klik + Leeg.
  • 2.Zijvenster rechts → Pagina's → + → Volledig ingesloten pagina.
  • 3.Geef de pagina een naam → klik Gereed.
  • 4.Klik op Insluiting toevoegen → Code insluiten → HTML plakken → Invoegen.
  • 5.Klik Publiceren → kies Beperkt — alleen mensen in je organisatie (Don Bosco-schooldomein).
  • 6.Kopieer het adres → deel via Smartschool.

Meerdere apps? Herhaal stap 2 per app. Google Sites maakt automatisch een navigatiemenu bovenaan.

Stap 02

Publiceren via GitHub Pages

Gratis webhost, professioneel adres. Voor wie wat verder wil gaan.

  • 1.Maak gratis account op github.com.
  • 2.+ New repository → naam (bv. quiz-biologie) → Public → Create.
  • 3.Add file → Create new file → naam exact index.html.
  • 4.HTML van Gemini plakken → Commit changes.
  • 5.Settings → Pages → Deploy from branch → main → Save.
  • 6.Na 1 minuut: gebruikersnaam.github.io/quiz-biologie

Afbeeldingen toevoegen: upload jpg/png via Add file → Upload files. Gebruik ze in HTML als <img src="naam.jpg">
